# 08 - Portfolio Positioning

## Como este projeto funciona como portfolio

EB2-NIW Readiness Studio e um portfolio tecnico completo por combinar engenharia de software aplicada, responsabilidade de privacidade e arquitetura orientada a produto.

## Competencias demonstradas

### Python
- Estruturacao de regras de negocio claras e testaveis.

### Flask
- Backend web leve com renderizacao server-side.

### SQLite
- Persistencia local-first, simples e portavel.

### IA plugavel
- Contrato unico para Mock, OpenAI, Azure OpenAI e Ollama.

### Arquitetura local-first
- Decisao tecnica com foco em controle de dados e custo.

### i18n
- Suporte PT-BR e EN-US desde a fundacao.

### Testes
- Harness com pytest cobrindo regras criticas.

### Documentacao spec-driven
- Contexto, especificacao, dominio, arquitetura, plano, fases e ADRs.

### Privacidade
- Regras explicitas de nao versionar dados sensiveis e nao hardcodar secrets.

### Impacto social/profissional
- Ajuda profissionais a organizar narrativa tecnica com transparência e responsabilidade.

## Uso como base para artigos no LinkedIn

- Arquitetura local-first para apps com IA.
- Como construir contratos de providers sem lock-in.
- Como separar software educacional de aconselhamento juridico.
- Como documentacao spec-driven acelera colaboracao com Copilot.
